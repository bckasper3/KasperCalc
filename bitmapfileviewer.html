Bitmap Editor

Bitmap editor for black and white OLED-style images

Size: 128×64 Cursor: — Byte addr: — Bit: — Pixel index: — Set pixels: 0
Import — paste C/C++ array, XBM, PBM, or drop .bmp/.bin file
Export

New creates a blank canvas at the selected size. ↻ 90° rotates the image clockwise. ⇆ Flip X and ⇅ Flip Y mirror horizontally or vertically. ⊡ Invert toggles every pixel. Clear sets everything to black. ↩ Undo / ↪ Redo step through your edit history (also Ctrl+Z / Ctrl+Y).

Tool selects what happens when you click or drag on the canvas. Draw sets pixels white. Erase sets them black. Fill flood-fills from the clicked pixel. Line and Rect draw previews while dragging and commit on release. Keyboard shortcuts: D draw, E erase, F fill, L line, R rect.

Zoom scales up the canvas for easier editing. The grid appears at 4× and above.

Byte Layout controls how C array bytes are decoded on import and encoded on export. This is the most common source of a scrambled image when pasting code from a library. Horizontal MSB is used by Adafruit GFX and most general-purpose libraries — the leftmost pixel is bit 7. Horizontal LSB is the XBM format — the leftmost pixel is bit 0. Vertical LSB is the SSD1306 page format used by most OLED Arduino libraries — bytes run column by column within 8-pixel tall pages, bit 0 is the top pixel. Vertical MSB is the same page layout but with bit 7 at the top. Changing this selector automatically updates the Export format to match.

Import accepts a dropped or browsed file (.bmp, .bin, .pbm, .xbm, .h, .c, .txt), or you can paste a C array, XBM, PBM P1, or raw hex bytes into the text box and click Parse. For C arrays without explicit width/height comments, the parser strips the array-size bracket (e.g. img[1024]) and then tries to match common display sizes (128×64, 128×32, 96×64, etc.) before falling back to a square-root guess. Adding a comment like // width: 96 height: 48 anywhere in the pasted text forces those dimensions.

Export converts the current canvas to the selected format. C array horizontal MSB and LSB output row-by-row. C array vertical LSB and MSB output in 8-pixel page columns for OLED controllers. XBM is a C header format used by Linux and some graphics tools. PBM P1 is a plain ASCII bitmap. Raw hex is space-separated byte values. Binary strings show each row as 1s and 0s. Click Export to generate, then Copy to copy to clipboard.

You can also drop a supported file directly onto the canvas, and draw with a stylus or touch on supported devices.
